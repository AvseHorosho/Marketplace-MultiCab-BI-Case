# Multi-Marketplace BI Governance & Reporting Protocol v1.0

## 1. Назначение документа

Документ фиксирует:
- порядок снятия показаний с маркетплейсов
- регламент версионности
- разделение технического и управленческого слоя
- правила freeze
- формат консолидированной отчётности

Применяется для WB / OZON / YM.

---

## 2. Уровни отчетности

### Level 1 — TDP (Technical Data Pack)
- Машиночитаемые CSV
- Rows / Cols контроль
- Snapshot_Date
- Freeze_Level
- Без интерпретаций

### Level 2 — Executive Report (T+3)
- Управленческий язык
- Оценка динамики
- Предварительные выводы

### Level 3 — CFO Final Snapshot (T+7)
- Консолидированный PnL
- Unit Economics
- After Ads
- Risk Matrix
- Capital Allocation

---

## 3. Версионность

Принцип:
- T+3 — оперативный
- T+7 — финансовый freeze
- Старые версии не перезаписываются
- Новые версии добавляются ниже старых

## 3.1 Обязательные атрибуты версии

- Snapshot_Date

- Period_Start / Period_End

- Freeze_Level (T+3 / T+7 / Monthly)

- Source_Data_Pack

- Ответственный аналитик

Без указания этих атрибутов версия считается невалидной.

---

## 4. Правило Freeze

Snapshot считается финальным, если:
- зафиксирована дата снятия
- подтвержден период
- проведён контроль строк
- сформирован MultiCab MASTER слой
- Проверена арифметическая целостность PnL MASTER
- Проверено совпадение сумм между вкладками (6 → 7 → 17)
- Зафиксирован режим фильтрации витрины

Иначе Freeze считается предварительным.
---

## 5. Разделение слоев

TDP ≠ CFO Report

TDP хранит данные.
CFO интерпретирует.

---

## 6. Обработка изменений показателей

Если показатели изменились:
1. фиксируется причина
2. классифицируется тип изменения
3. формируется новая версия

6.1 Классификация расхождений

A — операционное изменение
B — закрытие цикла
C — изменение параметров модели
D — корректировка источника
E — методическая ошибка

Без классификации изменение не считается проанализированным.

---

## 7. Консолидированная логика мультикабинета

Отчёт считается мультикабинетным, если:
- есть единый период
- есть PnL MASTER
- есть Ads MASTER
- есть Unit Economics MASTER
- есть Executive вывод
- Все площадки приведены к единому периоду. Используется единая структура индексации (см. стандарт нумерации)
- Все управленческие выводы трассируются к TDP
